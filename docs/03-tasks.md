# Task Backlog

## 1. Task Rules

- Every task must have one primary owner agent.
- Every task must be small enough to review independently.
- Every task must name expected files or directories before implementation.
- Every task must include acceptance criteria.
- Do not mix unrelated product, architecture, data, UI, strategy, and risk changes.
- Do not modify business code during documentation-only tasks.
- Business-code changes should run `npm.cmd run typecheck` and `npm.cmd run build` when feasible.
- The buy observation pool is not a new page or a separate product area. It is the fund strategy state layer inside the existing "我的观察" page.
- Industry watch items remain opportunity-tracking items and do not enter staged-buy-plan logic.
- Watchlist strategy upgrades must preserve the current add/remove watchlist flow.
- AI evidence interpretation is an evidence layer only. It may parse announcements, fund reports, industry news, policy events, and manual imports into structured evidence, but it must not directly create a buy plan or bypass risk vetoes.
- All `可进入分批买入计划草稿` states must come from system strategy conclusions. The conclusion is generated by deterministic rules first, with AI evidence added later as an input, and still never means automatic trading.
- My portfolio holdings should support lightweight current valuation refresh for held funds only, with a target interval of about one minute while the portfolio page is active.

## 2. Active Task Sequence

The next approved upgrade track is Intraday Decision Assist. This track adds a manual-review, rule-based intraday helper for deciding whether a held or watched fund needs plan execution review, add-position review, reduce-position review, or no action today. It is not an automatic trading system and must not connect to broker execution.

## Task 50: Define Intraday Risk Vetoes And Pause Conditions

Owner Agent: Risk Agent

Goal:

Define hard and soft risk controls for intraday add/reduce review.

In Scope:

- Hard vetoes:
  - data stale or unavailable
  - concentration exceeds personal limit
  - QDII/overseas exposure exceeds limit
  - risk event invalidates thesis
  - severe overheat or gap-up chase risk
- Soft warnings:
  - high volatility
  - liquidity or fee uncertainty
  - unresolved news conflict
  - data source mismatch
- Required wording for risk-blocked conclusions.

Out of Scope:

- Stop-loss automation.
- Forced liquidation instructions.

Expected Files:

- `docs/01-prd.md`
- `docs/02-tech-spec.md`
- `docs/03-tasks.md`

Acceptance Criteria:

- Every intraday buy-related output has risk checks and pause conditions.
- Sell/reduce language is framed as manual复核, not automatic sell instruction.
- Risk blocked state cannot be overridden by AI evidence.

Verification:

- Risk documentation review.
