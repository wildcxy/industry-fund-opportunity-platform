# 产品默认决策记录

## 1. 文档目的

记录当前版本已经明确的产品与技术默认值，减少后续评审、设计和工程搭建时的重复决策。

## 2. 当前默认值

### 产品方向

- 产品名称：行业基金机会捕捉平台
- 产品阶段：V1
- 产品形态：网页前端演示版
- 产品定位：帮助用户发现行业机会、理解逻辑并映射到相关基金的辅助决策平台

### 文档与沟通

- PRD 语言：偏正式中文汇报版
- 面向对象：产品、设计、前端、架构和管理层评审方

### 首版范围

- 包含：首页、行业详情页、基金发现页、基金对比页、我的观察页
- 包含：Mock 数据、指标口径、风险提示、页面交互与信息架构
- 不包含：真实交易、登录注册、个性化推荐、后台管理、实时数据接入

### 技术方向

- 默认前端框架：Next.js
- 默认语言：TypeScript
- 默认样式：Tailwind CSS
- 默认数据方式：静态 Mock 数据与本地 API Route
- 默认设备策略：桌面端优先，兼顾移动端响应式

### 架构原则

- 路由按页面能力拆分
- 模块按业务域组织
- 指标展示必须保留口径说明
- 风险提示为固定必备模块
- 前端与未来真实数据接入保持低耦合

## 3. 后续使用建议

1. 产品经理更新需求时，应优先同步本文件中的默认值是否需要调整。
2. 架构师输出工程方案时，应以本文件与 PRD 为共同依据。
3. 若后续进入真实数据接入阶段，应新增版本记录，不直接覆盖 V1 默认边界。
