# 个人持仓驱动的基金决策辅助系统 V1

## 1. 定位

本模块是行业基金机会捕捉平台的下一阶段核心能力，目标是把用户自己的基金持仓转化为可解释、可更新、可追踪的决策辅助视图。

系统不连接真实交易账户，不执行买卖，不输出强指令式投资建议。它只提供信息整理、组合体检、候选观察和风险提示，帮助用户更清楚地理解自己的基金持仓。

## 2. 用户主链路

默认链路为：

1. 用户导入支付宝或其他平台的基金持仓截图。
2. 系统生成 OCR/人工校对草稿，用户确认后形成持仓快照。
3. 系统按基金名称匹配真实基金代码。
4. 系统将基金加入真实候选池并触发盘后数据采集。
5. 系统用最新净值、前一日净值和昨日涨跌增强持仓。
6. 系统更新总持仓、市值、昨日盈亏、累计盈亏。
7. 系统生成组合体检、候选观察基金和决策辅助提示。

## 3. 核心模块

### 3.1 持仓导入

支持两类输入：

- 持仓快照：基金名称、持有金额、昨日收益、持有收益、持有收益率。
- 净值明细：基金代码、基金名称、持有份额、成本净值。

V1 默认先采用“图片上传 + OCR 草稿 + 人工确认”的安全流程。自动识别结果不得直接覆盖持仓数据，必须先进入确认草稿。

### 3.2 基金匹配与真实采集

系统按基金名称或基金代码搜索真实基金目录，并将匹配结果写入候选池。匹配后触发单只基金真实数据采集，采集范围包括：

- 基金主数据。
- 最新净值与前一日净值。
- 昨日涨跌。
- 近 1/3/6/12 月收益。
- 最大回撤与波动率。
- 基金规模与成立年限。
- 费率与赎回规则。
- 披露持仓。

如果采集失败，页面必须显示失败原因或“数据待补全”，不得使用演示数据覆盖。

### 3.3 每日持仓更新

每日盘后更新口径：

- 昨日涨跌：基金最新净值相对前一日净值的变化。
- 昨日盈亏：用户持有份额乘以净值变化；若只有截图快照，则优先展示截图中的昨日收益。
- 总持仓：所有持仓市值之和。
- 累计盈亏：截图持有收益或基于成本净值估算的累计收益。

QDII 可以存在 T+1 或更晚更新，必须展示数据日期和延迟提示。

### 3.4 组合体检

组合体检至少包含：

- 主题集中度：例如 AI、半导体、北证、资源、海外科技等方向占比。
- QDII 占比：提示海外市场、汇率和更新时间差异。
- 单只基金集中度：识别单只持仓金额占比过高的情况。
- 浮盈浮亏分布：识别高浮盈、深浮亏和接近成本区间。
- 同主题重复：识别多个基金高度相似的情况。
- 数据完整度：识别缺少净值、费率、持仓披露或成本规则的基金。

### 3.5 候选观察基金池

候选基金来源：

- 已持仓基金的同主题替代基金。
- 同主题中费率更低、规模更稳定、回撤更可控的基金。
- 与当前组合主题暴露互补的基金。
- 长期持有成本更清晰的基金。
- 数据完整度更高的基金。

候选池输出字段应包含：基金名称、基金代码、候选来源、候选理由、关键指标、数据质量、风险提示、是否可加入观察或对比。

### 3.6 决策辅助提示

允许输出的提示类型：

- 继续观察。
- 可加入候选观察池。
- 数据不足，暂不判断。
- 主题集中度偏高。
- QDII 占比偏高。
- 短期涨幅过快，需警惕拥挤风险。
- 回撤较深，需确认行业基本面与基金风格是否变化。
- 同主题重复度较高，建议进行横向比较。

禁止输出：建议买入、建议卖出、必须加仓、必须减仓、必涨、保证收益、抄底成功。

## 4. 数据与接口方向

### 4.1 前端视图状态

- `snapshot`：来自截图或手动金额录入。
- `matched`：已匹配基金代码，但未完成采集。
- `collecting`：正在采集真实数据。
- `enhanced`：已完成净值增强。
- `partial`：部分字段缺失。
- `failed`：匹配或采集失败。

### 4.2 后端数据域

后续实现应规划以下数据域：

- 用户持仓快照。
- 基金匹配结果。
- 基金采集任务。
- 持仓估值快照。
- 组合体检快照。
- 候选观察基金快照。
- 决策辅助提示快照。

### 4.3 数据源策略

数据源优先级：

1. AKShare 优先跑通基金目录、净值、费率和披露持仓。
2. 天天基金/东方财富作为基金主口径，可先由 AKShare 间接封装，后续抽象为独立 Provider。
3. Tushare 暂不作为基金净值主链路，未来用于股票、指数、行业估值和持仓穿透增强。

### 4.4 稳定性要求

- 单只基金失败不影响整批持仓更新。
- 每只基金记录任务状态、失败原因、尝试次数和最后成功日期。
- 原始响应按交易日缓存，便于重跑和审计。
- 页面显示数据日期、更新时间和数据质量。

## 5. 验收标准

1. 用户可以从支付宝截图或手动录入形成持仓快照。
2. 系统可以把持仓基金匹配到真实基金代码，并触发真实数据采集。
3. 页面能区分截图快照、净值增强、待采集、采集失败和数据待补全。
4. 页面能展示总持仓、昨日盈亏、累计盈亏和数据更新时间。
5. 系统能输出组合体检与候选观察基金，但不输出买卖指令。
6. 所有提示都包含理由、指标证据、数据质量和风险提示。

## 6. 默认边界

- 不接真实交易账户。
- 不做自动买卖。
- 不做收益承诺。
- 不把候选观察基金等同于买入建议。
- 不把行业热度等同于值得买。
- 不把截图金额当作基金真实净值数据。

## 7. V2 决策辅助口径

本阶段将“持仓调整策略建议”升级为证据驱动的复盘系统。系统不再只看短期涨跌，而是把以下证据合并判断：

- 基金自身表现：昨日涨跌、近 1 月、近 3 月、近 6 月收益。
- 风险指标：最大回撤、波动率。
- 组合权重：单只基金仓位、主题仓位、QDII/海外占比。
- 资产意图：长期持有型资产与高波动弹性主题分开判断。
- 行业证据：行业趋势、资金强度、估值性价比、行业 20 日表现。
- 数据质量：净值是否增强、行业证据是否缺失、费用和持仓披露是否待补。

### 7.1 决策等级

- `继续观察`：证据不足或未触发明显风险，仅提示继续补数据和跟踪。
- `短期波动`：单日或短期波动触发提醒，但近 1/3/6 月或行业证据仍有支撑，不等于趋势变坏。
- `趋势仍有支撑`：基金中期表现或行业证据仍偏强，适合按原计划复盘，不因单日净值扰动频繁操作。
- `阶段转弱`：近 1 月或近 3 月走弱，并伴随高波动、高回撤、仓位较高或行业证据缺失。
- `趋势破坏`：基金多周期走弱，同时行业趋势、资金或估值证据也偏弱，才进入更强风险提示。

### 7.2 长期持有与高波动主题分层

长期持有型资产包括 QDII、全球、海外、美股、纳斯达克、标普、亚洲、新兴市场等方向。它们默认不因单日净值波动直接触发减仓提示，重点观察海外市场趋势、汇率、估值和基金风格是否漂移。

高波动主题包括半导体、CPO、AI 算力、北证、有色、高端装备等方向。它们允许短期弹性更大，但必须设置仓位上限和回撤观察阈值。若短期连续上涨，系统会提示可能存在回调风险；若中期走弱且回撤扩大，系统会提示优先控仓复盘。

### 7.3 策略输出边界

系统可以输出“优先复盘”“暂缓加仓”“设置控仓阈值”“继续观察趋势是否变坏”“补齐行业证据后再判断”等辅助提示。

系统不能输出“立即买入”“必须卖出”“一定反弹”“确定见顶/见底”等交易指令或收益承诺。

### 7.3.1 操作信号分层

为了让策略更直观，系统新增“操作信号”，但它仍然是决策辅助，不是无条件交易指令。

- `分批配置观察`：中期证据仍有支撑、买入观察分数较高、仓位不高，可进入分批配置观察池。
- `买入观察`：趋势未明显破坏，可继续观察是否出现更好的估值、回调或费用条件。
- `回调观察`：短期有波动但中期证据未坏，适合等待回调确认，不追单日涨跌。
- `不追高，等回调观察`：高波动主题短期涨幅过高，重点提示回调风险。
- `继续持有观察`：更适合长期持有或海外资产，不因单日净值扰动频繁调整。
- `暂停加仓`：阶段趋势转弱，先观察 1 到 2 周能否修复。
- `减仓复盘`：多周期趋势和行业证据同时偏弱，优先复盘是否降低仓位。
- `数据不足，先补证据`：缺少净值、行业、费用或持仓证据，不做强判断。

每个操作信号必须同时展示：

- 买入观察分数。
- 信号强度。
- 触发理由。
- 证据缺口。
- 非投资建议提示。

### 7.4 页面解释要求

每只持仓基金的复盘卡片必须展示：

- 决策等级。
- 置信度。
- 仓位占比。
- 昨日、近 1 月、近 3 月、最大回撤、波动率。
- 证据摘要。
- 证据缺口。
- 对应资产意图和行动提示。

如果行业证据缺失，系统必须降低置信度，并明确说明“本次判断不会把短期涨跌直接升级为趋势破坏”。
